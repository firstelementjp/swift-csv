# 🔧 設定

このページは v0.9.8 で実装されている Swift CSV の現在の動作を説明します。

## インポート設定

### CSV 形式

- 最初の行はヘッダーとして扱われます
- `ID` カラムは必須です
- 新規投稿には空の `ID` 値を使用します
- 既存投稿の更新時のみ既存の `ID` を使用します
- 一般的な CSV 形式の区切り文字は自動検出されます
- CSV 囲み文字は標準のダブルクォート形式です

### サポートされるヘッダータイプ

- `post_title`, `post_content`, `post_status` などの標準 WordPress 投稿フィールド
- `cf_` プレフィックスを使用するカスタムフィールド
- `category`, `post_tag`、カスタムタクソノミー名などのタクソノミーフィールド

### 複数値とタクソノミールール

- 複数値フィールドは `|` を使用します
- 階層タクソノミー値は `>` を使用します
- タクソノミー解析とカスタムフィールド解析は自動的に処理されます

## エクスポート設定

### エクスポートオプション

- **投稿タイプ**: エクスポートする投稿タイプを選択
- **投稿ステータス**: ステータスでエクスポート行をフィルター
- **範囲**: 基本フィールド、すべてのフィールド、またはフックによるカスタム出力をエクスポート
- **エクスポート制限**: オプションの制限；`0` は無制限を意味
- **プライベートメタ**: オプションでプライベートメタキーを含める

### 出力形式

- UTF-8 CSV 出力
- 標準カンマ区切り
- 標準 CSV エスケープ
- BOM 出力なし

## 自動バッチ処理

Swift CSV はバッチサイズを自動的に調整します。

### インポート

- インポートバッチは自動的に計算されます
- 現在の実装では環境制限に応じてバッチあたり約 **10-50 行** を目標とします

### エクスポート

- エクスポートバッチは自動的に計算されます
- 現在の実装ではデータセットサイズに応じてバッチあたり **500-2000 投稿** を使用します

### なぜこれが重要か

これにより以下を削減できます：

- リクエストタイムアウト
- メモリスパイク
- 大規模な単一リクエスト処理負荷

## UI 動作

インポートとエクスポート中、管理 UI は以下を提供します：

- リアルタイム進捗更新
- ジョブ実行中の処理詳細
- WordPress 翻訳によるローカライズされたインターフェーステキスト

## ライセンスと Pro 統合

Swift CSV Free と Swift CSV Pro は WordPress フックと権限チェックを通じて統合されています。

利用可能な動作：

- Pro 対応管理メッセージング
- Pro 機能が利用可能な場合のライセンス有効化 UI
- オプションの Pro 動作用の機能フラグ

## 現在の実用上の制限

- 非常に大きなファイルは依然としてサーバーリソースに依存します
- 区切り文字検出は自動ですが、不正な CSV は依然として失敗する可能性があります
- エクスポート形式は CSV に固定されています
- インポート動作は WordPress スタイルのフィールド名とプラグイン規約を前提としています

## 開発者拡張性

Swift CSV には以下のためのフックベースの拡張ポイントが含まれています：

- エクスポートヘッダーと行
- インポート権限と検証
- インポートフィールド準備
- バッチサイズ調整
- 機能フラグと管理アクセス制御

現在のフックリファレンスについては [開発者向けフック](hooks.md) を参照してください。
