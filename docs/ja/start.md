# 🚀 はじめに

Swift CSV v0.9.9 へようこそ。このガイドは、インストールから最初の成功したインポートとエクスポートまでの最短経路をカバーします。

## Swift CSV の機能

Swift CSV は、管理インターフェースから WordPress コンテンツを CSV としてインポートおよびエクスポートするのに役立ちます。

以下をサポートしています：

- 標準 WordPress 投稿フィールド
- カスタム投稿タイプ
- `cf_` プレフィックス付きカスタムフィールド
- 階層値を含むタクソノミーカラム
- 大規模データセットの自動バッチ処理
- 英語ベース文字列と日本語翻訳のローカライズされた管理 UI

## 管理画面の場所

有効化後、以下を開きます：

- **ツール → Swift CSV**

## 最初のインポート

1. **ツール → Swift CSV → インポート** を開く
2. 対象の投稿タイプを選択
3. CSV ファイルをアップロード
4. 既存投稿を更新するかどうかを選択
5. 安全にファイルを検証したい場合は、まずドライランを実行
6. インポートを開始して完了を待つ

### 最小テスト CSV

```csv
ID,post_title,post_content,post_status
"","サンプル投稿","これはコンテンツです","publish"
"","別の投稿","さらにコンテンツ","draft"
```

注：

- `ID` カラムは必須です。
- 新規投稿には空の `ID` 値を使用します。
- 投稿を更新する場合のみ既存の投稿 ID を使用します。
- 可能であればファイルを UTF-8 で保存してください。

### インポート CSV ルール

- 最初の行にはヘッダーが含まれている必要があります。
- カスタムフィールドは `cf_` プレフィックスを使用する必要があります。
- 複数値フィールドは `|` 区切りを使用します。
- 階層タクソノミー値はレベル間に `>` を使用します。

## 最初のエクスポート

1. **ツール → Swift CSV → エクスポート** を開く
2. 投稿タイプを選択
3. 投稿ステータスフィルターを選択
4. エクスポート範囲を選択
5. 必要に応じてエクスポート制限を設定
6. エクスポートを開始して生成された CSV をダウンロード

## 表示される内容

処理中、Swift CSV は以下を表示します：

- リアルタイム進捗更新
- アニメーション付きプログレス UI
- 現在処理中のアイテムごとのログ詳細
- ジョブ完了時の完了状態

## 一般的な CSV パターン

### カスタムフィールド

```csv
ID,post_title,cf_price,cf_color,cf_tags
"","商品 A","19.99","赤","セール|注目"
"","商品 B","29.99","青","新着|注目"
```

### 階層タクソノミー

```csv
ID,post_title,category,post_tag
"","技術投稿","テクノロジー > WordPress > プラグイン","技術|wordpress|php"
```

## バッチ処理

バッチ処理を手動で設定する必要はありません。

Swift CSV は以下に基づいて処理サイズを自動的に調整します：

- データセットサイズ
- サーバー実行制限
- メモリ制約
- インポートまたはエクスポートコンテキスト

現在の動作は [設定](config.md) で文書化されています。

## 最初の確認

最初のインポートまたはエクスポート後、以下を確認してください：

- 投稿が期待通りに作成または更新された
- カスタムフィールドが正しく保存された
- タクソノミーが正しく割り当てられた
- エクスポートされたカラムが期待されるヘッダーと一致する
