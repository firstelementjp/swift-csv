# 🐛 トラブルシューティング

現在の Swift CSV 実装の一般的な問題と解決策。

## クイックチェック

1. `WP_DEBUG` と `WP_DEBUG_LOG` を有効にする
2. ブラウザコンソールで JavaScript エラーを確認
3. まず小さな CSV でテスト
4. CSV ヘッダーが期待される形式と一致することを確認
5. **ツール → Swift CSV** を使用していることを確認

## 一般的な問題

### インポートが直ちに失敗する

以下を確認：

- CSV に `ID` カラムが含まれている
- 最初の行にヘッダーが含まれている
- ファイルが有効な CSV である
- 選択された投稿タイプが正しい
- Nonce または権限チェックがカスタムコードによってブロックされていない

### 新規投稿が作成されない

新規投稿を作成する場合：

- `ID` カラムを保持する
- 新規行の `ID` 値を空にする
- 新規投稿にダミーの数値 ID を使用しない

### 既存投稿が更新されない

更新の場合：

- 実在する WordPress 投稿 ID を使用する
- インポート UI で既存投稿更新オプションを有効にする
- 選択された投稿タイプが既存投稿と一致することを確認

### カスタムフィールドがインポートされない

以下を確認：

- カスタムフィールドヘッダーが `cf_` で始まっている
- 複数値カスタムフィールドが `|` を使用している
- カンマを含む場合、CSV 値が正しく引用符で囲まれている

### タクソノミー値が正しくインポートされない

以下を確認：

- 階層タクソノミー値が `>` を使用している
- 複数のタクソノミー値が `|` を使用している
- タクソノミーが選択された投稿タイプに存在する

### 進捗 UI が更新されない

以下を確認：

- 管理 JavaScript が正しく読み込まれた
- ブラウザコンソールに致命的なエラーがない
- AJAX リクエストが正常に完了している
- 移動する前にリクエストサイクルが完了するまで待機した

### 大きなファイルが遅く感じる

Swift CSV はすでに自動バッチ処理を使用しています。

処理がまだ遅い場合：

- まず小さなデータセットでテスト
- サーバーメモリと実行制限を確認
- サブセットのみが必要な場合はエクスポート制限を削減
- 完全処理前にドライランを使用してインポートファイルを検証

### テキストが文字化けする

以下を確認：

- CSV を UTF-8 で保存
- `mbstring` が有効であることを確認
- WordPress/データベース文字セットが `utf8mb4` であることを確認
- ローカライズされた UI テキストが期待される場合、翻訳ファイルが存在することを確認

### Pro ライセンス UI が正しく表示されない

以下を確認：

- Pro 機能を期待する場合、Swift CSV Pro がインストールされている
- Swift CSV Pro が有効化されている
- ライセンス関連のサーバー設定が有効である

## デバッグログ

必要に応じて `wp-config.php` に追加：

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

便利なチェック：

```bash
wp core version
php -v
wp plugin status swift-csv
tail -f wp-content/debug.log | grep "Swift CSV"
```

## 問題を報告する際

以下を含めてください：

- WordPress バージョン
- PHP バージョン
- Swift CSV バージョン
- ブラウザとバージョン
- 正確なエラーメッセージ
- 可能であれば小さなサンプル CSV
- インポート、エクスポート、またはその両方で問題が発生するか
