# 📋 変更履歴

## [0.9.9] - 2026-04-06

### 🚀 インポートの信頼性とパフォーマンス

- **大規模インポートのメモリ使用量削減** - インポートバッチが CSV データを増分的に読み取り、メモリ内に完全な CSV コンテンツを保持する代わりにキャッシュされたファイルオフセットを再利用するようになりました
- **重複 CSV 解析の削減** - バッチ処理が同じバッチ内で解析された行データを再利用し、繰り返しの解析オーバーヘッドを削減します
- **改善されたバッチ診断** - インポートプロファイリングログで、大規模インポートのデバッグ時に時間とメモリのホットスポットを特定しやすくなりました

### 🛠️ CSV 解析とインポート UI の修正

- **RFC 4180 準拠の CSV 解析** - コア `str_getcsv()` 使用がバックスラッシュエスケープを無効にし、引用符で囲まれたフィールドがより予測可能に解釈されるようになりました
- **インポートログの安定性改善** - インポートログタブがより確実に初期化され、完了時に最近のログが利用可能になり、エラーログがエラーカウントと一致するようになりました
- **より安全なログクリーンアップ動作** - UI ログクリアが現在のインポートパネルをより正確に対象とするようになりました

### 📦 リリースワークフローの更新

- **ブランチ対応リリーススクリプト** - `test-release.sh` が現在のブランチまたは明示的に提供されたブランチからリリース ZIP をビルドできるようになりました
- **自動アーティファクトクリーンアップ** - 一時的な `test-release` 出力と生成された最小化アセットがローカルリリースビルド後に削除されます
- **クリーンなリポジトリ状態** - 生成されたリリースアーティファクトと最小化された配布ファイルが Git で追跡されなくなりました
