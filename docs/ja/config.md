# 🔧 設定

このページでは FE CSV Import & Export の現在の動作を説明します。

## インポート設定

### CSV 形式

- 最初の行はヘッダーとして扱われます
- `ID` カラムは必須です
- `ID` カラムに値がある場合、既存の投稿の上書きを行います
- CSVファイルから新規投稿を作成するには `ID` 列を空欄にします
- 区切り文字はカンマ、セミコロン、タブのいずれかを自動検出します
- CSV 囲み文字は標準のダブルクォート形式です

### サポートされるヘッダータイプ

- `post_title`, `post_content`, `post_status` などの標準 WordPress 投稿フィールド（postsテーブルのカラム名）
- `cf_` で始まるキー列はカスタムフィールドとして扱います（例： cf_price は「price」というカスタムフィールドとしてインポート）
- `category`, `post_tag`、カスタムタクソノミー名などのタクソノミー名

### 複数値とタクソノミールール

- １つのカラムに複数の値を登録するには `|` で区切ります（ `|` の前後の半角スペースの有無は機能に影響しません）
- タームの階層の指定が必要な場合は `>` を使用します（ `>` の前後の半角スペースの有無は機能に影響しません）
- タクソノミーは、term_id または name どちらで出力するか選択できます

## エクスポート設定

### エクスポートオプション

- **投稿タイプ**: エクスポートする投稿タイプを選択
- **投稿ステータス**: 公開記事、またはすべてのステータスか、エクスポートする投稿を絞り込み
- **範囲**: 更新頻度の高い基本フィールド、すべてのフィールド、またはフックによる出力カラムの選択・並び順を指定可能
- **エクスポート制限**: オプションの制限；`0` は無制限を意味
- **プライベートメタ**: オプションでプライベートメタキー（アンダースコアから始まるキー）を含める

### 出力形式

- UTF-8 CSV 出力
- 標準カンマ区切り
- 標準 CSV エスケープ
- BOM 出力なし

## 自動バッチ処理

FE CSV Import & Export はサーバーの環境設定値（max_execution_time, memory_limit等）に応じてバッチサイズを自動的に調整します。

### インポート

- インポートのバッチサイズは自動的に計算されます
- 現在の実装では環境制限に応じてバッチあたり約 **10-50 行** を目標とします
- 近日中にエクスポートと同様の効率化により、インポート時間の短縮を図ります

### エクスポート

- エクスポートバッチは自動的に計算されます
- 現在の実装ではデータセットサイズに応じてバッチあたり **500-5000 投稿** を使用します

### なぜこれが重要か

これにより以下を削減できます：

- リクエストタイムアウト
- メモリスパイク
- 大規模な単一リクエストによる処理負荷

## UI 動作

インポートとエクスポート中、管理 UI は以下を提供します：

- リアルタイム進捗更新
- ジョブ実行中の処理詳細
- インポート時、新規・更新・エラーに分類して処理済みの投稿の概要を表示

## ライセンスと Pro 統合

Free版とPro版は連携して動作します。Pro版がインストールされている場合、以下の機能が利用可能になります：

- ライセンスキーの有効化・無効化 UI
- Pro版限定機能の利用（Advanced Custom Fields対応・UpdraftPlus連携・実行パスワードの設定　他）

## 現在の実用上の制限

- 非常に大きなファイルは依然としてサーバーリソースに依存します
- 区切り文字検出は自動ですが、不正な CSV は依然として失敗する可能性があります
- エクスポート形式は CSV に固定されています
- インポートするCSVファイルのヘッダー行は WordPress データベースのカラム名（post_title, post_content など）およびカスタムフィールド名を正確に記述する必要があります

## 開発者拡張性

FE CSV Import & Export には、以下のためのアクション/フィルターフックがあります：

- CSVヘッダーのカスタマイズと各行データの加工（エクスポート）
- エクスポートするカスタムフィールド項目を決定するためのサンプル記事の指定（デフォルトでは一行目の記事を元に出力するカスタムフィールドを決定）
- インポート実行前の権限チェックとデータ検証
- インポート前のフィールド値変換・加工
- 一度に処理する行数（バッチサイズ）のカスタマイズ
- Pro機能の有効化や管理画面に設定項目を追加する

現在のフックの詳細については [開発者向けフック](ja/hooks.md) を参照してください。
