# 💡 例

このページは現在の FE CSV Import & Export 実装の実用的な CSV 例を示します。

## 管理画面の使用フロー

### インポート

1. **管理ダッシュボード → ツール → FE CSV Import & Export → インポート** を開く
2. 対象の投稿タイプを選択
3. CSV ファイルをアップロード
4. 必要に応じて更新と Dry Run オプションを選択
5. インポートを開始

### エクスポート

1. **管理ダッシュボード → ツール → FE CSV Import & Export → エクスポート** を開く
2. 投稿タイプを選択
3. 投稿ステータスとエクスポート範囲を選択
4. 必要に応じてエクスポート制限を設定
5. エクスポートを開始し、完了後、 CSV をダウンロード

## CSV 形式の基本

### 最小インポートファイル

```csv
ID,post_title,post_content,post_status
"","サンプル投稿","これはコンテンツです","publish"
"","別の投稿","さらにコンテンツ","draft"
```

注：

- `ID` カラムは必須です。
- 新規投稿には空の `ID` を使用します。
- 既存投稿の更新時のみ既存の ID を使用します。

### カスタムフィールド

```csv
ID,post_title,cf_name,cf_email,cf_tags
"","山田太郎","太郎","taro@example.com","開発者|wordpress|php"
"","田中花子","花子","hanako@example.com","デザイナー|ui|ux"
```

注：

- カスタムフィールドヘッダーは `cf_` で始まる必要があります。
- 複数値カスタムフィールドは `|` を使用します。

### 階層タクソノミー

```csv
ID,post_title,category,post_tag
"","技術投稿","テクノロジー > WordPress > プラグイン","技術|wordpress|php"
"","デザイン投稿","デザイン > UI > Web","デザイン|ui|ux"
```

注：

- 階層レベルには `>` を使用します。
- 複数のタクソノミー値には `|` を使用します。
- ターム名でインポートする場合、該当するタームが存在しない場合は新しく作成されます。

## 一般的なシナリオ

### ブログ移行

```csv
ID,post_title,post_content,post_date,post_status,cf_original_id
"","私の最初のブログ投稿","ここにコンテンツ","2026-01-15 10:00:00","publish","legacy-123"
"","私の2番目の投稿","さらにコンテンツ","2026-01-16 10:00:00","draft","legacy-124"
```

このシナリオは以下のような場合に適しています：

- 元の投稿IDを保持したままコンテンツを移行したい場合
- 大量のコンテンツをバッチで移行する場合
- インポート前にドライランでテストを行いたい場合

### 商品カタログ

```csv
ID,post_title,post_content,post_status,cf_price,cf_sku,cf_stock,cf_color,cf_size
"","クラシックTシャツ","プレミアムコットンTシャツ","publish","19.99","TS-001","50","黒|白|グレー","S|M|L"
"","デニムジーンズ","クラシックフィットデニムジーンズ","publish","49.99","DJ-001","25","青","30|32|34"
```

このシナリオは以下のような場合に適しています：

- 複数の商品属性を一括でインポートしたい場合
- カスタムフィールドにリスト形式でデータを保存したい場合
- 外部ツール用に商品データをエクスポートしたい場合

### チームディレクトリ

```csv
ID,post_title,cf_name,cf_title,cf_email,cf_department,cf_skills
"","山田太郎","シニア開発者","taro@company.com","エンジニアリング","php|wordpress|javascript"
"","田中花子","UXデザイナー","hanako@company.com","デザイン","ux|ui|figma"
```

このシナリオは以下のような場合に適しています：

- 構造化されたプロフィールデータを管理したい場合
- スキルなどのデータをパイプ区切りで保存したい場合
- レビュー用に内部ディレクトリをエクスポートしたい場合

## エクスポート指向の例

典型的なエクスポートには、WordPressのコア項目に加えてオプションのカスタムフィールド列とタクソノミー列が含まれます。

```csv
ID,post_title,post_content,post_status,category,post_tag,cf_price
"101","サンプル投稿","これはコンテンツです","publish","テクノロジー > WordPress","技術|プラグイン","19.99"
```

## 検証のヒント

インポート前に：

- ファイルを UTF-8 で保存
- 最初の行をヘッダーとして保持
- 最初のカラムとして `ID` を含める
- 手動でCSVを作成する場合、値にカンマが含まれる可能性がある場合は引用符を使用（Googleスプレッドシート等のツールからエクスポートする場合は自動的に処理されます）
- 小さなテストファイルから開始
