# 📦 インストール

## WordPress 管理画面からインストール

1. WordPress 管理画面にログイン
2. **プラグイン** → **新規追加** に移動
3. 「Swift CSV」を検索
4. **今すぐインストール** をクリック
5. **有効化** をクリック

## 手動インストール

1. [最新版 (v0.9.9) をダウンロード](https://github.com/firstelementjp/swift-csv/releases/download/v0.9.9/swift-csv-v0.9.9.zip)
2. ダウンロードした ZIP ファイルを展開
3. `swift-csv` フォルダを `/wp-content/plugins/` にアップロード
4. WordPress 管理画面の **プラグイン** に移動
5. 「Swift CSV」を見つけて **有効化** をクリック

## 動作環境

### システム要件

- **WordPress**: 6.6 以降（推奨: 6.6+）
- **PHP**: 8.1 以降（推奨: 8.1+）
- **メモリ制限**: 128MB 以上（大規模 CSV 処理用）
- **拡張機能**: `mbstring`, `zip`（ファイル処理用）

### サーバー推奨

- **アップロード最大ファイルサイズ**: 64MB 以上
- **POST 最大サイズ**: 64MB 以上
- **最大実行時間**: 300 秒以上
- **出力バッファリング**: 有効（大規模ファイル処理用）

### ブラウザ互換性

- **最新ブラウザ**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **JavaScript**: 有効（AJAX 処理に必須）
- **CSS アニメーション**: サポート（プログレスバー アニメーション用）

## 初期設定

### 有効化後

1. **プラグインにアクセス**: WordPress 管理メニューの **ツール** → **Swift CSV** に移動
2. **設定を構成**: デフォルトの投稿タイプとエクスポート制限を設定
3. **テストインポート**: まず小さな CSV ファイルのインポートを試す
4. **テストエクスポート**: 数件の投稿をエクスポートして機能を確認
5. **進捗を確認**: 処理中に進捗 UI を監視

### 基本設定

1. **デフォルト投稿タイプ**: デフォルトで作業する投稿タイプを選択
2. **エクスポート制限**: エクスポートごとの最大投稿数を設定（0 = 無制限）
3. **バッチ処理**: すべてのデータセットで自動（適応: 10-50 インポート行、500-2000 エクスポート投稿）
4. **デバッグモード**: トラブルシューティングのために `WP_DEBUG` を有効化

### 高度な機能

- **進捗追跡**: インポートとエクスポート中のリアルタイム進捗
- **バッチ処理**: 適応的バッチサイズ（10-50 インポート行、500-2000 エクスポート投稿）
- **ローカライズされたインターフェース**: 英語ベース文字列と日本語翻訳
- **ライセンス統合**: Pro 版ライセンス管理
- **複数値フィールド**: パイプ区切り値サポート（例: "値1|値2|値3"）

## インストールのトラブルシューティング

### 一般的な問題

**WordPress ディレクトリでプラグインが見つからない場合:**

- `swift-csv` フォルダが `/wp-content/plugins/` にあることを確認
- フォルダ権限を確認（755 推奨）
- すべてのファイルが完全にアップロードされていることを確認

**有効化に失敗する場合:**

- PHP バージョンの互換性を確認（8.1+ 必須）
- 必要な PHP 拡張機能がインストールされていることを確認
- WordPress バージョンの互換性を確認（6.6+ 必須）

**インポート/エクスポート中のメモリエラー:**

- `wp-config.php` で PHP メモリ制限を増やす: `define('WP_MEMORY_LIMIT', '256M');`
- バッチ処理を有効化（すべてのデータセットで自動）
- サーバーのアップロード制限を確認

**プログレスバーが動作しない場合:**

- ブラウザコンソールで JavaScript エラーを確認
- CSS ファイルが正しく読み込まれていることを確認
- まず小さなデータセットでテスト
- プラグインの競合を確認

**日本語文字が表示されない場合:**

- CSV ファイルが UTF-8 で保存されていることを確認
- データベース文字セットが `utf8mb4` であることを確認
- 翻訳ファイルが正しく読み込まれていることを確認

**ライセンス有効化の問題:**

- Swift CSV Pro がインストールされ有効化されていることを確認
- ライセンスサーバーの設定を確認
- 適切なメッセージ表示のために翻訳ファイルを確認
- 有効なライセンスキーでテスト

## パフォーマンス最適化

### 大規模データセットの場合

**自動最適化:**

- **インポートバッチ**: サーバーパフォーマンスに基づき 10-50 行
- **エクスポートバッチ**: データセットサイズに基づき 500-2000 投稿
- **メモリ管理**: 自動クリーンアップ、ストリームされた CSV バッチ読み込み、より効率的なバッチ処理
- **リアルタイム更新**: シマーアニメーション付きプログレスバー

`v0.9.9` の最近のインポート改善により、同じバッチ内での繰り返し CSV 解析作業が削減され、管理 UI に表示される完了ログの信頼性が向上しています。

**手動設定は不要**

プラグインは以下に基づいてバッチサイズを自動的に最適化します：

- サーバー実行時間制限
- 利用可能なメモリ
- データセットサイズ
- 処理の複雑さ

### サーバー設定

**.htaccess 設定**（必要な場合）:

```apache
# アップロード制限を増やす
php_value upload_max_filesize 64M
php_value post_max_size 64M
php_value max_execution_time 300
php_value memory_limit 256M

# 出力バッファリングを有効化
php_flag output_buffering on
php_value output_buffering 4096
```

**php.ini 設定**（アクセス可能な場合）:

```ini
upload_max_filesize = 64M
post_max_size = 64M
max_execution_time = 300
memory_limit = 256M
output_buffering = 4096
```

## 最初のステップ

### テストインポート

1. **テスト CSV を作成**:

```csv
ID,post_title,post_content,post_status
"","テスト投稿 1","これはテストコンテンツです","publish"
"","テスト投稿 2","さらにテストコンテンツ","draft"
```

**注**: `ID` カラムは必須です。新規投稿には空値（`""`）を、更新には実際の投稿 ID を使用してください。

2. **インポート処理**:
    - **ツール → Swift CSV → インポート** に移動
    - 投稿タイプを選択
    - テスト CSV をアップロード
    - プログレスバー アニメーションを監視
    - 結果を確認

### テストエクスポート

1. **エクスポート処理**:
    - **ツール → Swift CSV → エクスポート** に移動
    - 投稿タイプを選択
    - エクスポート制限を設定（5 投稿を試す）
    - **CSV をエクスポート** をクリック
    - 投稿タイトル付きリアルタイム進捗を監視
    - CSV ファイルをダウンロードして確認

### 機能の確認

- **プログレスバー**: 処理中のアニメーション付きリアルタイム進捗
- **リアルタイム詳細**: 個別の投稿タイトルが表示
- **ローカライズされたインターフェース**: 自然言語メッセージ（英語/日本語）
- **バッチ処理**: 大規模データセットで自動
- **複数値サポート**: パイプ区切り値でテスト
